# 개발 일정 (2026.07 ~ 2026.10)

7월에 하드웨어 bring-up과 수동 제어 루프를 완성하고, 8월에는 PM·열화상·영상 기반 위험 판단 모듈을 병렬적으로 통합합니다. 9월까지 **MVP End-to-End 데모**(정상 조리 → 사용자 부재 → 연기·열 위험 증가 → 후드 강풍 → 인덕션 차단 시뮬레이션)를 완성하고, 이후 10월까지 반려동물·물체 접근, 음성 명령, fail-safe, UI, 로그 재생 기능을 반복 확장·통합합니다. **매월 측정 결과를 기반으로 임계값과 제어 정책을 보정**합니다.

## MVP 정의

| MVP | 목표 | 관련 모듈 |
|---|---|---|
| MVP 1 | Pi 4 → ESP32 수동 명령으로 팬·LED·부저 제어 루프 확립 | `pi4/comm/`, `esp32/` |
| MVP 2 | SmokeScore 기반 후드 풍량 자동 조절 | `pizero/sps30/`, `pi4/smoke/` |
| MVP 3 | MLX90640 기반 HeatScore·과열 판단 | `pi4/thermal/` |
| MVP 4 | 사용자 존재/부재 판단 (AbsentConfirmed) | `pi4/vision/` |
| MVP 5 | 비조리 중 반려동물·물체 접근 시 인덕션 잠금 | `pi4/vision/`, `esp32/` |
| MVP 6 | 웨이크워드 + 제한 명령어 음성 제어 | `pi4/voice/` |
| E2E | 복합 위험 시나리오 통합 데모 | `pi4/fusion/` + 전체 |
