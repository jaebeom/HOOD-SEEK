# 하드웨어 구성

## 장비 목록

| 구분 | 장비 | 역할 |
|---|---|---|
| 판단 노드 | Raspberry Pi 4 | 영상·열화상·음성·PM 융합 판단, 위험도 산출 |
| 센서 게이트웨이 | Raspberry Pi Zero | SPS30 수집·로그화, Pi 4 watchdog |
| 액추에이터 MCU | ESP32 | 후드 팬 PWM, 인덕션 차단 시뮬레이션, 잠금 LED, 부저 |
| 미세입자 센서 | Sensirion SPS30 | PM1.0 / PM2.5 / PM4 / PM10 측정 |
| 열화상 카메라 | MLX90640 | 조리면 온도 분포 (I2C) |
| 일반 카메라 | V4L2 호환 USB 카메라 | 사용자 존재·반려동물·물체 탐지 |
| 마이크 | ReSpeaker Lite USB 2-Mic | 제한 명령어 음성 입력 |
| 액추에이터 | 후드 팬(PWM), 인덕션 차단 시뮬레이터, 부저, LED | 제어 대상 |
| 전원 | 전원 공급형 USB 허브 | Pi 4 주변 USB 장치 전원 안정화 |

## 배치·배선 주의사항

- **MLX90640**: I2C 배선 길이를 **짧게 유지** (신호 무결성)
- **SPS30**: 기름이나 수증기가 **직접 응축되지 않는 위치**에 배치, 후드 흡입부 근처에서 Pi Zero가 수집
- **Pi 4 USB 전원**: 카메라 + ReSpeaker + 기타 USB 장치가 많으므로 반드시 **전원 공급형 USB 허브** 사용
- **UART 배선**: 핵심 제어 링크이므로 배선 불량에 주의, heartbeat로 상시 검증

## Fail-safe 하드웨어 구조

```
        ┌─────────┐  watchdog  ┌─────────┐
SPS30 ──> Pi Zero ├───────────>│  Pi 4   │
        └────┬────┘            └────┬────┘
             │ Pi4 정지 감지         │ UART (heartbeat + 명령)
             │                      v
             │                 ┌─────────┐
             └── fail-safe ───>│  ESP32  ├──> 팬 / 차단 / LED / 부저
                 트리거          └─────────┘
```

Pi 4가 멈추거나 통신이 끊기면 Pi Zero와 ESP32가 fail-safe 상태로 전환합니다. 최종 동작의 안전 판단(timeout, 명령 거부)은 항상 ESP32가 수행합니다.
