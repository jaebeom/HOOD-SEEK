# 테스트 시나리오

`tests/` 디렉토리는 아래 5개 시나리오를 기준으로 구성합니다.

## 평가지표 (6가지)

1. 연기 농도 감지
2. 사용자 재실 여부 확인
3. 실시간 과열 모니터링
4. 반려동물 식별
5. 음성 명령어 인식
6. 제어 반응 속도

## 시나리오

### TS-1. PM 기반 자동 후드 풍량 조절

정상 조리 중 PM 농도 증가에 따라 후드 풍량이 **약풍 → 중풍 → 강풍**으로 단계적으로 조절되는지 확인한다.

- 검증 포인트: SmokeScore 산출, 이동평균·히스테리시스에 의한 풍량 진동 억제

### TS-2. 사용자 부재 및 과열 위험 판단

조리 중 사용자가 자리를 비운 상태에서 조리면 과열이 감지될 경우 **위험 단계가 상승**하고 **경고 및 인덕션 차단**이 수행되는지 확인한다.

- 검증 포인트: AbsentConfirmed 판정 시간, HeatScore 기반 단계 전이 (MEDIUM → HIGH)

### TS-3. 복합 위험 상황 End-to-End 제어

사용자 부재, 연기 증가, 과열이 **동시에** 발생하는 복합 위험 상황에서 **후드 강풍, 인덕션 차단, 부저 및 LED 경고**가 정상 동작하는지 확인한다.

- 검증 포인트: CRITICAL 전이 조건, Pi 4 → ESP32 명령 전달 및 집행 지연

### TS-4. 비조리 중 반려동물 접근 제한

비조리(Idle) 상태에서 반려동물 또는 미확인 물체가 인덕션 위험 구역에 접근할 경우 **인덕션 잠금 상태가 유지**되는지 확인한다.

- 검증 포인트: ROI 침입 판정, ON/화력 증가 명령 거부

### TS-5. 음성 명령 및 안전 우선순위 검증

제한 음성 명령으로 후드와 인덕션이 제어되는지 확인하고, **위험 상태에서는 안전 제어가 음성 명령보다 우선**되는지 확인한다.

- 검증 포인트: 웨이크워드·명령어 인식률, HIGH 이상에서 "인덕션 켜"/"화력 올려" 거부

## 성능 목표 (온디바이스)

| 기능 | 목표 지연 | 모델 크기 |
|---|---|---|
| 연기량 증가 판단 (SmokeIndex) | ≤ 10ms | 없음 (규칙 기반) |
| 사용자 존재 판단 | 100~300ms | 5~15MB |
| 이상 가열 탐지 | ≤ 50ms | 없음 (규칙 기반) |
| 위험도 판단 (상태기계) | ≤ 10ms | 없음 (규칙 기반) |
| 반려동물·물체 탐지 | 150~500ms | 5~12MB |
| 음성 명령 인식 | 0.5~2s | 50~150MB 또는 경량 KWS |
